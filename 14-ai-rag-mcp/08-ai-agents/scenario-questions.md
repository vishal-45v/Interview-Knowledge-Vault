# Chapter 08: AI Agents — Scenario Questions

1. You're building a customer support agent for a SaaS product. The agent needs to: look up customer accounts, check subscription status, process refunds (with approval), create support tickets, and escalate to human agents. A customer with a complex billing dispute could require 15+ tool calls in a single session. Design the complete agent architecture: memory management, tool set, approval workflows, escalation logic, and context window management.

2. Your ReAct agent is stuck in an infinite loop: it calls `search_web`, gets results, decides it needs more information, calls `search_web` again with a slightly different query, and repeats this 20+ times without generating a final answer. What are the three root causes of this failure mode, and how would you modify the system prompt, the tool, and the agent loop to detect and break out of this cycle?

3. You need to build a multi-agent pipeline for processing insurance claims. The pipeline involves: (a) an extraction agent that reads PDFs and extracts claim data, (b) a verification agent that checks against a policy database, (c) a risk assessment agent that scores the claim, and (d) an approval agent that makes the final decision. Design the multi-agent coordination: what framework, how data flows between agents, how to handle failures mid-pipeline, and how to maintain an audit trail.

4. You're designing a code generation agent that writes Python functions, runs them in a sandbox, and iterates based on test results. The sandbox execution takes 3-10 seconds per run. The agent sometimes generates an infinite loop in the generated code. Design the complete system: sandbox architecture, timeout handling, iteration limits, success criteria, and how to communicate compilation/runtime errors back to the agent in a format it can reason about.

5. A team wants to deploy an agent that autonomously manages cloud infrastructure: it can create EC2 instances, modify security groups, and delete resources based on cost optimization rules. The agent will run on a schedule without human oversight. What are the specific risks you would enumerate to the team, what HITL controls would you implement, what blast radius limits would you enforce at the tool level, and how would you design the rollback mechanism?

6. You're building a research agent that browses the web, extracts information from multiple pages, synthesizes findings, and generates a report. After running for 2 minutes, the agent's context window fills up because it's accumulating tool call histories. How would you implement context management: what do you summarize, what do you discard, and how do you ensure the agent doesn't lose track of the goal?

7. You have an agent that uses 8 different tools. In production, you notice that the agent frequently calls `search_internal_docs` before `lookup_customer_by_id`, even though for most queries the customer lookup should come first. This is a tool selection ordering problem. How do you diagnose it, and what are the three different approaches to fix it (prompt-level, tool description level, and architecture level)?

8. A financial firm wants to deploy a portfolio management agent that monitors 50 client portfolios in parallel. Each portfolio check involves: 3 database queries, 2 external API calls, and one LLM reasoning step. They want results in under 5 minutes for all portfolios. Design the parallelization strategy: which components run in parallel, how you handle rate limiting across simultaneous API calls, and how you aggregate results without losing individual portfolio context.

9. Your multi-agent system (LangGraph-based) has two agents that both believe they have "ownership" of a task and make conflicting changes to the same shared resource (a document). This is a concurrent write conflict. How would you redesign the state management in LangGraph to prevent this? Describe the specific state schema changes and the edge conditions you would add.

10. You're asked to evaluate three agent frameworks for a new project: LangGraph, AutoGen, and CrewAI. The project is a semi-automated data analysis pipeline where analysts provide partial direction but the agent fills in the gaps. Build a framework evaluation matrix covering: multi-agent support, human-in-the-loop, observability, code execution, deployment complexity, and community maturity. Give a final recommendation with justification.
