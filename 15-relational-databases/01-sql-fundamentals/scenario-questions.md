# Chapter 01 — SQL Fundamentals: Scenario Questions

Real-world production scenarios requiring deep SQL knowledge to diagnose and solve.

---

1. Your e-commerce platform has an `orders` table with 50 million rows. A product
   manager wants a report: "For each customer, show their most recent order date,
   total lifetime spend, and number of orders — but only for customers who have
   placed at least 3 orders and have spent more than $500." The query a junior
   engineer wrote takes 45 seconds. Walk through how you would write this query
   correctly and efficiently.

2. You have a `users` table and an `events` table. A stakeholder asks: "Show me
   all users who have NEVER triggered the 'purchase' event." A colleague wrote
   a LEFT JOIN with WHERE events.type IS NULL, but it's returning wrong results
   for users who have purchase events AND other event types. What is wrong, and
   how do you fix it?

3. You're building a "people you may know" feature. You have a `friendships`
   table with `(user_id, friend_id)`. Write a query to find users who are friends
   of friends but not already direct friends. What SQL technique is core to
   solving this, and what edge cases must your query handle?

4. A finance report needs a running total of daily revenue grouped by region,
   plus a grand total row, plus subtotals per region — all in a single result
   set that can be exported to Excel. The report was previously generated by
   three separate queries merged in application code. How do you consolidate
   this into a single SQL query?

5. Your `products` table has a `tags` column stored as a comma-separated string
   (legacy design). You need to query all products that have the tag 'electronics'.
   The current query uses LIKE '%electronics%' and is slow. You cannot change
   the schema today. How do you make this query correct and as fast as possible
   within the constraint?

6. You are investigating a support ticket: "Customer ID 9821 says their order
   history is missing 3 orders." You have `orders`, `order_items`, and `products`
   tables. When you query joining all three, those 3 orders don't appear. What
   JOIN-related issue might explain this, and how do you write a diagnostic query
   to find the root cause?

7. A batch job runs every night at midnight and inserts 500,000 rows into an
   `audit_log` table. It uses INSERT ... SELECT from multiple joined tables and
   takes 20 minutes. The INSERT is not inside a transaction (autocommit). A
   colleague suggests wrapping it in a single transaction. Another suggests
   batching with LIMIT. Which approach do you take and why?

8. You need to implement a leaderboard that shows a player's score, their rank,
   the rank of the person above them, and the score gap between them. The
   leaderboard has 2 million players. The product wants it refreshed every
   30 seconds. How do you write the SQL and what structural decisions do you make?

9. A reporting query joins `sales`, `products`, `categories`, and `regions`
   tables and runs a GROUP BY with SUM and COUNT. The result set has 5,000 rows
   and takes 8 seconds. EXPLAIN shows a sequential scan on `sales` (200M rows)
   and a hash join with `products`. The sales table has indexes on `product_id`
   and `sale_date`. How do you diagnose why the indexes are not being used?

10. You have a multi-tenant SaaS database where every table has a `tenant_id`
    column. A developer accidentally wrote a query without a WHERE tenant_id = ?
    filter in a complex subquery inside a view. The outer query does filter by
    tenant_id, so they assume it is safe. Is it actually safe? Under what
    conditions would this produce data leakage, and how do you prevent this
    class of bug structurally?
