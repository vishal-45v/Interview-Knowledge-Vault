## Rails Version Notes

### Rails 7.0 (January 2022)
- **Hotwire** (Turbo + Stimulus) became the default front-end stack, replacing Webpacker
- **Import maps** introduced as default JS bundling (no Node.js required)
- **ActiveRecord encryption** added natively
- `config.active_record.encryption` DSL
- **Zeitwerk** became the only supported autoloader (Classic mode removed)

### Rails 7.1 (October 2023)
- **Dockerfile** generated by default (`rails new` includes Docker support)
- **solid_queue** introduced as a DB-backed ActiveJob adapter
- **solid_cache** introduced as a DB-backed cache store
- **Composite primary keys** supported natively
- `ActiveRecord::Base.normalizes` for attribute normalization
- `config.active_record.automatically_invert_plural_associations`
- `authenticate_by` added to `has_secure_password`
- **Puma** default thread count changed; `RAILS_MAX_THREADS` default = 3

### Rails 7.2 (August 2024)
- **Dev containers** configuration added
- Ruby 3.1+ required
- `config.assume_ssl` middleware added
- `config.yjit` defaults to enabled when YJIT available
- `bin/setup` improvements

### Rails 6.x → 7.x Migration Points (Know for Interviews)
- Webpacker → import maps / jsbundling-rails / cssbundling-rails
- Classic autoloader → Zeitwerk (constant reloading semantics changed)
- `belongs_to` is required by default (since Rails 5, but still trips people up)
- `config.load_defaults 7.0` behaviour flags

## Quick Reference: Most-Tested Rails Topics (2024–2025)

1. N+1 queries — detection, prevention with `includes`/`eager_load`/`preload`
2. Sidekiq idempotency and retry behaviour
3. Devise customization (custom fields, controllers, routes)
4. Rails credentials vs environment variables
5. Hotwire / Turbo Streams for real-time updates
6. Pundit policy scopes
7. `before_action` filter chains and `return` placement
8. `update` vs `update!` vs `update_attribute` — which skips validations/callbacks
9. Database connection pool exhaustion
10. Zero-downtime deployments with Kamal or Capistrano
