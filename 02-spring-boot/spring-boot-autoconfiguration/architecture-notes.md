# Spring Boot Auto-Configuration — Architecture Notes

---

## Auto-Configuration Loading Mechanism

```
@SpringBootApplication
        │
        ▼
@EnableAutoConfiguration
        │
        ▼
AutoConfigurationImportSelector.selectImports()
        │
        ▼
SpringFactoriesLoader.loadFactoryNames()  (Spring Boot < 2.7)
OR
ImportCandidates.load()  (Spring Boot 2.7+)
        │
        ▼
Returns list of ~150 AutoConfiguration class names
        │
        ▼
Each class evaluated against current environment:
  @ConditionalOnClass   → classpath check
  @ConditionalOnBean    → context check
  @ConditionalOnProperty → property check
  @ConditionalOnMissingBean → absence check
        │
        ▼
Matching configurations applied → beans registered
```

---

## Condition Evaluation Order

```
Conditions are evaluated in this order:

1. @ConditionalOnClass / @ConditionalOnMissingClass
   (cheapest — just classpath lookup)

2. @ConditionalOnBean / @ConditionalOnMissingBean
   (requires BeanFactory scan)

3. @ConditionalOnProperty
   (requires Environment access)

4. @ConditionalOnExpression (SpEL)
   (most expensive — SpEL evaluation)

Custom @Conditional annotations are evaluated when encountered.
```

## META-INF Files Reference

```
src/main/resources/META-INF/

├── spring/
│   └── org.springframework.boot.autoconfigure.AutoConfiguration.imports
│       (Spring Boot 2.7+ auto-configurations, one class per line)
│
├── spring.factories
│   (Legacy format, still supported in Spring Boot 3)
│   org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
│     com.example.MyAutoConfiguration
│
├── additional-spring-configuration-metadata.json
│   (IDE hints for custom @ConfigurationProperties)
│
└── spring-configuration-metadata.json
    (Generated by spring-boot-configuration-processor)
```

## @ConditionalOnClass Safety

```java
// SAFE: @ConditionalOnClass uses ASM to read bytecode, NOT Class.forName()
// The referenced class can safely be absent from the classpath.

@ConditionalOnClass(name = "com.amazonaws.services.s3.AmazonS3")
// Uses string-based class name lookup — no ClassNotFoundException

@ConditionalOnClass(AmazonS3.class)
// Uses class literal — ONLY safe because the annotation processor
// translates it to string form before evaluation.
// At runtime, if AmazonS3 is not present, this is still safe.
```
